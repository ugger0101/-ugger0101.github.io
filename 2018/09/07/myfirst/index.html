Docker概述

Docker为啥出现？

环境配置麻烦，费时费力。

Java – apk – 发布

Java – jar（环境） — 打包项目带上环境（镜像） – Docker（仓库：商店） – 下载镜像直接运行

Docker通过隔离机制，可以将服务器利用到极致

本制：所有技术都是因为出现了一些问题，我们需要去解决

Doceker历史

vm: linux centos    隔离，需要打开多个虚拟机
docker：隔离、镜像（最核心环境4M+jdk+mysql）十分小巧，运行镜像就可以了！ 小巧！ 几个M秒级启动

必须掌握

Docker

Go语言开发！开源项目

官方地址：https://www.docker.com/

官方文档：https://docs.docker.com/ 文档很详细

仓库地址：https://hub.docker.com/

Docker能干嘛

之前虚拟机技术

缺点：

  1. 资源占用多
  2. 冗余步骤多
  3. 启动慢

容器化技术

比较Docker和虚拟机

DevOps(开发、运维)

应用更快速的交付和部署

更便捷的升级和扩缩容

更简单的系统运维

更高效的计算资源利用

Docker是内核级别的虚拟化，可以在一个物理机上运行很多的容器实例！服务器的性能可以被压榨到极致
